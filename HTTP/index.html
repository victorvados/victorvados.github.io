HTTP

El Protocolo de Transferencia de HiperTexto (Hypertext Transfer Protocol) es un sencillo protocolo cliente-servidor que articula los intercambios de información entre los clientes Web y los servidores HTTP. La especificación completa del protocolo HTTP 1/0 está recogida en el RFC 1945. Fue propuesto por Tim Berners-Lee, atendiendo a las necesidades de un sistema global de distribución de información como el World Wide Web.


Desde el punto de vista de las comunicaciones, está soportado sobre los servicios de conexión TCP/IP, y funciona de la misma forma que el resto de los servicios comunes de los entornos UNIX: un proceso servidor escucha en un puerto de comunicaciones TCP (por defecto, el 80), y espera las solicitudes de conexión de los clientes Web. Una vez que se establece la conexión, el protocolo TCP se encarga de mantener la comunicación y garantizar un intercambio de datos libre de errores.


HTTP se basa en sencillas operaciones de solicitud/respuesta. Un cliente establece una conexión con un servidor y envía un mensaje con los datos de la solicitud. El servidor responde con un mensaje similar, que contiene el estado de la operación y su posible resultado. Todas las operaciones pueden adjuntar un objeto o recurso sobre el que actúan; cada objeto Web (documento HTML, fichero multimedia o aplicación CGI) es conocido por su URL.

Página Web

  • Consta de objetos como por ejemplo pueden ser :
    • Documento HTML
    • Imagen GIF
    • Applet Java
    • Etc...
  • Ejemplo Una página Web con cinco imágenes tienes seis objetos. [Documento HTML + 5 imágenes]
  • El documento HTML referncia a los otros objetos

URL

  • Corresponde a la dirección de la página web
  • Tiene dos componentes:
    • El nombre del host del servidor
    • La ruta de los objetos
  • Ejemplo www.iesvirgendelcarmen.com/informatica
  • Host www.iesvirgendelcarmen.com
  • Ruta /informatica

Navegador y Servidor

  • Navegador es el agente de usuario o cliente que solicita la página web
  • Ejemplos Internet Explorer, Mozilla Firefox...
  • Servidor web Alberga los objetos

Características HTTP

  • HTTP define como los clientes web demandan las páginas web
  • Y como el servidor transfiere dichas páginas a los clientes
  • En el navegador ponemos IP o nombre de la máquina
  • http://www.google.es:80 por defecto asume el puerto 80 , es válido también www.google.es
  • El servidor, normalmente, manda la página index.html, index.htm, index.php... junto a los objetos que presente dicho documento HTML
  • Existen los protocolos HTTP/0.9 (obsoleto), HTTP/1.0 y HTTP/1.1 (versiones actuales) y HTTP/2.0 (nuevo)
  • El servidor envia los objetos solicitados sin almacenar dato alguno sobre el cliente
  • Si en pocos segundo se solicita el mismo objeto el servidor NO responde que acaba de servir el objeto, lo vuelve a mandar
  • HTTP es un protocolo sin estado

Mensaje HTTP Respuesta

  • Línea inicial de status:
    • 200 OK Conexión con éxito
    • 400 Bad Request petición que NO puede ser atendida
    • 404 Not Found el objeto no existe en el servidor
  • Línea inicial de status:
    • Connection close: el servidor va a cerrar la conexión
    • Server: servidor web
    • Last-Modified: fecha y hora que el objeto fue creado o modificado por última vez
    • Content-Lenght número de bytes de objeto enviado
    • Content-type: tipo de objeto enviado
  • Línea de cuerpo: el objeto en cuestión

Cookies

  • Es un fragmento de información que se almacena en el disco duro del visitante de una página web a través de su navegador, a petición del servidor de la página
  • Esta información puede ser luego recuperada por el servidor en posteriores visitas
  • Las cookies tienen tres componentes::
    • Una línea de cabecera de cookie en el mensaje HTTP de petición
    • Un archivo de cookie que se almacena en el usuario y gestionado por el navegador
    • Una base de datos en el sitio web

Tipos de MIME

  • MIME (Multipurpose Internet Mail Extensions), (Extensiones de Correo Internet Multipropósito)
  • Especificaciones dirigidas a que se puedan intercambiar a través de Internet todo tipo de archivos (texto, audio, vídeo, etc.) de forma transparente para el usuario
  • El protocolo básico de transmisión de mensajes electrónicos de Internet soporta solo caracteres ASCII de 7 bit
  • Esto limita los mensajes de correo electrónico, ya que incluyen solo caracteres suficientes para escribir en un número reducido de lenguajes, principalmente Inglés
  • MIME define mecanismos para enviar otros tipos de información en un e-mail, incluyendo texto en lenguajes distintos del Inglés utilizando otros conjuntos de caracteres además de ASCII
  • MIME es adicionalmente un componente fundamental en protocolos de comunicación como HTTP

Protocolos Seguros

  • Secure Sockets Layer
  • Se usa en transacciones seguras vía Web
  • Habilita la posibilidad de usar firmas digitales, criptografía y resumen de mensajes
  • Proporciona:
    • Autenticación aseguramos que el usuario es el que dice ser
    • Integridad no hay variaciones en las trasacciones de archivos
    • Confidencialidad cifrado de mensajes
  • https HTTP sobre SSL

Firma electrónica

  • Asocia la identidad de una persona o de un equipo informático a un mensaje o documento
  • Además asegura la integridad del documento o mensaje
  • Pasos a seguir:
    • Calcular el hash o resumen del mensaje o archivo
    • Se cifra el hash con la clave privada (firma)
    • Se envía el mensaje o archivo con el hash cifrado.
    • En el otro lugar se quita la firma del mensaje o archivo
    • Se obtiene el hash enviado descifrando con la clave pública
    • Se calcula el hash del archivo o mensaje
    • Si coincide con el anterior, el mensaje no ha cambiado (integridad)
    • Si coincide, integridad además de autenticar al remitente
